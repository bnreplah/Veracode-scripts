#   The plan for the DAST framework is to break it down into microservices. Each can be used on their own to accomplish an individual ask, but together they can be used as a API framework and UI for using the APIs from the CLI.
#   The DAST framework will consist of the following features:
#      [] DAST Health check
#      [] DAST configuration health check ( if it conforms with best practices or not and recommendations)
#      [] DAST Scan health check ( Reading the status of the scan to determine next actions )
#      [] DAST findings
#      [] DAST finding break down
#      [] DAST Configurations
#      [] See all DAST scans
#      [] Configure a scan
#      [] Bulk block list
#      [] Bulk allows list
#      [] Login script helper
#      [] Set DAST configuration Variables
#      [] Set Login methods
#      [] Edit a scan
#      [] Delete a scan
#      [] Download report
# Future considerations:
# # DAST auto config
